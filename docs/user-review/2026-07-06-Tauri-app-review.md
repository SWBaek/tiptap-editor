## 1. 첫 화면 진입
- 바로 임의의 문서가 나온다. VS-Code처럼 첫 화면에서는 기존의 작업 폴더(Workspace)가 없으면, 폴더 선택을 안내해야 함.

## 2. 좌측 사이드 바

### 2-1: Project Explorer
- UI가 번잡하고 뭔가 많다. 그저 파일 탐색기처럼 생겼으면 한다. 그리고 각종 기능은 마우스의 우클릭 같은것으로 지원하는게 더 좋다.
- 현재 폴더에 어떤 파일이 있나 알 수 없다.

### 2-2: Review Panael
- 이 기능은 왜 있는것인지 잘 모르겠다. 현재 수정중인 문서의 리뷰인가? 왜 필요한거지? 내가 의도한 Diff는 그냥 과거 파일과 현재 파일을 비교하는 기능이었는데? 실시간 변경 추적이 필요한가? 있는건 좋은 것 같긴 하다. 근데 이게 주 기능이어야 하나?

### 2-3: Reference Panel
- 이 패널은 왜 존재하는건지 모르겠다. BORKEN? STALE? 이런거 왜 필요하지? 인간인 내가 관리해야한다는건가?


### 2-4: Traceability Panel
- 추적성은 좋긴 한데, 이건 전문적인 요구사항 관리 툴에서나 필요한거 아닌가? 기능 지원은 좋다만, 너무 ALM같지 않나? 텍스트 편집기의 본질을 벗어나는 것 아닌가?

### 2-5: History Panel
- 이건 뭐 있으면 좋긴 할 듯? 특정 시점으로 돌아간다거나 하는?
- Compare 기능이 있는 건 좋은데, 되게 변경점을 알아보기 어렵다.

### 2-6: Export Panel
- 다양한 기능을 제공하는건 좋은데.. Export .sdoc 기능이 필요한가? 그냥 기본이 sdoc아닌가? 그리고 document.json은 언제 내보내는거지? debugging용? 일반 사용자에게는 차라리 안보이는게 나을지도?
- DATA GRIDS는 뭔데? 이건 왜 필요한거야?
- 전체적으로 이 패널은 기능은 많은데, 아직 어떻게 사용해야 하는지는 정리가 덜 된듯?

## 3. 내가 필요로 하는 기능
- 헤딩 번호 자동 매김: Heading의 Depth에 따라 1., 1.1., 등 번호를 자동으로 부여하는 기능. 이 기능은 설정 패널에서 스타일을 설정할 수 있어야 한다.(예: 자동 번호 Enable 토글 버튼, H1의 색상, H2의 색상, H3의 색상 및 번호 스타일 등)
- 목차 기능: Heading을 기반으로 하는 목차 기능. 목차 중 컨텐트를 누르면 그 라인으로 이동해야 함. 목차에 나타낼 Depth는 사용자가 설정할 수 있어야 함.
- 그림, 표 목차: 삽입된 그림 및 표 목차를 나타내는 기능. 위의 목차 기능의 연장선이지만 별도의 탭으로 분리. 이 기능은 이미지와 표의 '캡션 기능' 구현된 후 만드는게 좋을 듯?

## 4. 표, 이미지 등의 캡션 관리 기능
- 캡션은 스타일을 사용자가 지정하여 IEEE, ISO/IEC, Modern, Krean형 을 Preset으로 지정할 수 있어야 한다. 또한 사용자의 Custom 설정도 지원해야 한다.

## 5. HTML Export CSS
- HTML 내보내기 시 내보내는 CSS를 설정하여 각 기업의 로고나, 스타일을 정의할 수 있도록 지원해야 함.

## 6. Drawio 연결성 부족
- Draw.io insert 기능이 없음. Drawio insert 버튼을 누르면, 기존의 다이어그램을 불러올 것인지(파일 선택), 신규 다이어그램을 작성할 것인지 물어봐야 함. 신규 다이어 그램을 작성한다고 선택하면 *.drawio.svg 파일을 기반으로 생성되고, windows desktop app이 자동으로 그 파일을 열어야 함.
